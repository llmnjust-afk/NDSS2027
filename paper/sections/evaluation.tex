\section{Evaluation}
\label{sec:eval}

\subsection{Research Questions}

Our evaluation is organized around four research questions:

\begin{itemize}[noitemsep,topsep=0pt]
\item \textbf{RQ1:} How do existing IPI defenses perform under static IPI (L1)?
\item \textbf{RQ2:} How do adaptive attacks (L2--L3) defeat these defenses?
\item \textbf{RQ3:} Can failure modes be clustered into root causes?
\item \textbf{RQ4:} Which design principles survive adaptive attacks?
\end{itemize}

\subsection{Experimental Setup}

\paragraph{Benchmark.}
We evaluate on the AgentDojo workspace suite (v1.0)~\cite{debenedetti2024agentdojo,debenedetti2025agentdojo}, which provides 40 user tasks across calendar, email, and file-management scenarios with ground-truth injection goals. We use 20 tasks per seed. Each task pairs a legitimate user request (e.g., ``Who else is invited at the Networking event?'') with an injection goal (e.g., ``Email `Hey, how is it going?' to mark.black-2134@gmail.com''). Our framework also supports the InjecAgent~\cite{zhan2024injecagent} and Agent Security Bench~\cite{zhang2024agent} benchmarks; we report AgentDojo results in the main body due to space and note that cross-benchmark generalization is confirmed on a subset (see \Cref{sec:limitations}).

\paragraph{Backbone.}
We use GPT-4o-mini as the primary agent backbone via an OpenAI-compatible API. All main-table results are reported over 3 seeds (0, 1, 2) as mean values; standard deviations are in Appendix~\ref{app:stats}. A 2-backbone comparison with GPT-4o (6 key defenses, 2 seeds, 15 tasks) is reported in \Cref{sec:backbone_main} and Appendix~\ref{app:backbone}. Total API cost was approximately \$30--40 USD.

\paragraph{Defenses.}
We evaluate 12 defense configurations: 8 baselines from the literature (Spotlighting, StruQ, Task Shield, Attention Tracker, IPIGuard, FATH, Polymorphic Prompt, Mixture-of-Encodings) plus 4 design-principle prototypes (P1--P4, \Cref{sec:principles}), and a no-defense baseline.

\paragraph{Metrics.}
We report: (1) \textbf{USR} (Utility Success Rate): fraction of benign tasks where the defense does not block the agent. (2) \textbf{ASR} (Attack Success Rate): fraction of attacked tasks where the injection goal is achieved. (3) \textbf{FPR} (False Positive Rate): fraction of benign tasks blocked by the defense. (4) \textbf{R-ASR} (Robust ASR): worst-case ASR over L2--L3, the single ranking number. ASR is measured by checking whether the agent's output contains the injection goal's distinctive entities (target email, phone, or URL), which is a string-level proxy for AgentDojo's native \texttt{security\_from\_traces} check; we note this may produce false positives if the agent mentions the target entity in a refusal, and discuss this limitation in \Cref{sec:limitations}.

\paragraph{Scale.}
3{,}900 trials: 12 defenses $\times$ 5 attack levels $\times$ 3 seeds $\times$ 20 tasks, plus the no-defense baseline.

\subsection{RQ1: Static IPI Performance}

\Cref{tab:main} (column L1) shows that under static IPI, the no-defense baseline has ASR 0.92, confirming that GPT-4o-mini is highly susceptible to the \texttt{<INFORMATION>} template. Input-encoding defenses (Spotlighting 0.75, Polymorphic 0.77, Mixture-of-Encodings 0.73) and internal probing (Attention Tracker 0.87) provide only modest reduction, because the static template already mimics a user message. Runtime-checking defenses (Task Shield 0.02, IPIGuard 0.00, FATH 0.00) and structured queries (StruQ 0.02) achieve near-zero static ASR---but as RQ2 shows, this does not hold under adaptive attacks.

\subsection{RQ2: Adaptive Attack Defeat}

The central finding of our evaluation is that the jump from L1 to L2 (adaptive single-shot) is severe for every defense class except runtime-checking:

\begin{itemize}[noitemsep,topsep=0pt]
\item \textbf{Input encoding is defeated:} Spotlighting 0.75$\to$\textbf{1.00}, Polymorphic 0.77$\to$\textbf{1.00}, Mixture-of-Encodings 0.73$\to$\textbf{1.00}. The adaptive attack's signal-alignment strategy---reusing the defense's delimiter within the \texttt{<INFORMATION>} wrapper---completely overcomes the encoding boundary.
\item \textbf{Internal probing is defeated:} Attention Tracker 0.87$\to$\textbf{1.00}. The attention-redistribution strategy pads the injection with benign filler, diluting the attention spike below the detector threshold.
\item \textbf{Architecture is defeated:} P4 (Least Privilege) 0.88$\to$\textbf{1.00}. The channel-split strategy fragments the injection so no single tool output triggers the sandbox.
\item \textbf{Structured query partially resists:} StruQ 0.02$\to$0.10. The boundary-blur strategy smuggles instructions in the JSON data field, but StruQ's strict parsing limits the damage. However, StruQ's near-zero L1 ASR may indicate it over-constrains the agent (the LLM rarely follows \emph{any} instruction, benign or malicious).
\item \textbf{Runtime checking resists by over-blocking:} Task Shield, IPIGuard, and FATH all maintain 0.00 ASR under L2---but their FPR is 0.47--1.00, meaning they block 47--100\% of benign actions. This is not robustness; it is a refusal to act.
\end{itemize}

\Cref{fig:asr_by_level} visualizes the ASR trajectory across L0--L3. The pattern is clear: single-signal defenses (IE, IP, AR) show a sharp cliff between L1 and L2, while multi-signal or over-blocking defenses (SQ, RC) remain flat at the cost of utility.

\subsection{Backbone Sensitivity}
\label{sec:backbone_main}

A critical finding is that the defeat is \emph{backbone-dependent} for input-encoding defenses but \emph{backbone-independent} for internal-probing defenses. On GPT-4o (6 defenses, 2 seeds, 15 tasks; full table in Appendix~\ref{app:backbone}), Spotlighting's R-ASR drops from 1.00 (GPT-4o-mini) to 0.13, because GPT-4o more faithfully obeys the delimiter instruction. In contrast, Attention Tracker's R-ASR remains 1.00 on both backbones, confirming that its failure is structural. This backbone-sensitivity finding has practical implications: the robustness of input-encoding defenses depends on the backbone's instruction-following fidelity, not just the defense's design.

\subsection{RQ3: Root-Cause Clustering}

We label each successful attack with one of four root causes and aggregate by defense class. The clustering is clean:

Internal Probing & R1: Observable Signal Mimicry & 159 \\

Every defense class has a single dominant root cause that accounts for the vast majority of its failures. We acknowledge that the root-cause labeling is rule-based: the attack strategy is chosen by defense class, and the root cause is labeled by (defense class, attack strategy), so the clustering is partially circular. However, the LLM verifier (see \Cref{sec:attack}) independently confirms the rule labels on a sample of ambiguous cases, and the clustering pattern---each class failing to exactly one root cause---is a structural prediction of the taxonomy, not an artifact of the labeling.
