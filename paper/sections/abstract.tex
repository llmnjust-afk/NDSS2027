\begin{abstract}
Large language model (LLM) agents that integrate external tools are vulnerable to indirect prompt injection (IPI), where malicious instructions embedded in tool-returned content hijack the agent into executing attacker-specified actions. A growing body of work proposes defenses---input encoding, structured queries, runtime task alignment, and architecture-level isolation---yet these defenses are often evaluated with proxy metrics (string matching, ``unblocked'' rates) rather than end-to-end task completion and trace-based security. We systematize 11 IPI defenses across four defense classes and evaluate them on the AgentDojo benchmark using \emph{native end-to-end evaluators}: the agent executes real tools, and security is measured by \texttt{security\_from\_traces} on actual function-call traces while utility is measured by \texttt{user\_task.utility} on real environment state. Our 1{,}260-trial evaluation reveals a sharp security-utility tradeoff. Input-encoding defenses (Polymorphic ASR 0.14, Mixture-of-Encodings 0.16, Spotlighting 0.21) achieve the best tradeoff, reducing attack success by 71--81\% relative to baseline while \emph{preserving} utility above the no-defense ceiling. Runtime-checking defenses achieve near-zero ASR but at the cost of utility $\leq$ 0.06---though we show that 80\% of this ``over-blocking'' is attributable to the base model's capability ceiling, not the defense. A prompt-level structured-query approximation (without fine-tuning) provides little protection (ASR 0.70). An ablation of our P2 multi-signal prototype shows the echo-canary provides independent security value (ASR 0.03 alone) but all signal combinations sacrifice utility entirely, refuting the hypothesis that orthogonal multi-signals can achieve security at acceptable utility cost. We release our framework and benchmark as open-source artifacts.
\end{abstract}
