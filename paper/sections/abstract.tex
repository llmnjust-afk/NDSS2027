\begin{abstract}
Large language model (LLM) agents that integrate external tools are vulnerable to indirect prompt injection (IPI), where malicious instructions embedded in tool-returned content hijack the agent into executing attacker-specified actions. A growing body of work proposes defenses---input encoding, structured queries, runtime task alignment, and architecture-level isolation---yet these defenses are typically evaluated only against static injection templates. Recent work by Zhan et al.\ showed that adaptive attacks, which tailor the injection to the defense's detection signal, break three representative defenses. We extend this line of inquiry: we systematize 10 faithfully-evaluated IPI defenses across five defense classes, build a unified adaptive attack framework that materializes a matched adversary for each class, and conduct a large-scale evaluation on the AgentDojo benchmark. Our results reveal that input-encoding defenses relying on a single observable signal are defeated by adaptive attack (ASR rises from 0.73--0.77 to 1.00 on GPT-4o-mini), while runtime-checking defenses resist at their default thresholds only by over-blocking (FPR 0.47--1.00), though Task Shield achieves a better operating point (FPR=0.07) at a lower threshold. We show this defeat is backbone-dependent: on GPT-4o, input-encoding defenses partially resist (R-ASR drops to 0.13) because the stronger backbone more faithfully obeys delimiter instructions. We propose a falsifiable empirical hypothesis about single-signal fragility, identify four root-cause families that cluster by defense class, and instantiate design-principle prototypes, finding that orthogonal multi-signal defenses are necessary but incur a utility cost. We release our framework and benchmark as open-source artifacts.
\end{abstract}
