\section{Supplemental Experiments}
\label{app:supplemental}

The prior version of this paper included supplemental experiments (unknown-defense settings, no-feedback settings, cross-defense transfer, threshold sweeps, and held-out defense validation) conducted under the string-matching proxy methodology. Because these proxy results \emph{contradict} our native end-to-end findings (e.g., proxy reported Spotlighting ASR=1.00; native shows ASR=0.21), we have removed them from this revision rather than present unsupported numbers. We summarize what was removed and what remains:

\paragraph{Removed (proxy-based, not re-run natively).} (1) Settings A/B/C (unknown-defense, no-feedback, cross-defense transfer)---all used proxy ASR. (2) Threshold sweep for Task Shield and Attention Tracker---used proxy FPR/USR metrics; the native pipeline does not use the same threshold mechanism. (3) Held-out defense validation via SecAlign---used proxy ASR on Mistral-7B. (4) Query/cost analysis referencing the 3{,}900-trial proxy evaluation.

\paragraph{Retained (native end-to-end).} (1) Main results table (\Cref{tab:main}): 11 defenses, 990 trials, native \texttt{security\_from\_traces} + \texttt{user\_task.utility}. (2) P2 ablation (\Cref{sec:p2_ablation}, \Cref{app:p2_ablation}): 270 trials, three configurations. (3) Over-blocking diagnosis (\Cref{app:overblock}): baseline comparison on identical task triples. (4) Statistical analysis (\Cref{app:stats}): Wilson CIs from native data.

\paragraph{Planned.} We plan to re-run threshold sweeps, cross-defense transfer, and held-out validation using the native pipeline in future work. The native pipeline infrastructure (\Cref{app:native}) supports these experiments; the constraint is API cost and time.
