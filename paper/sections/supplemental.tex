\section{Supplemental Experiments}
\label{app:supplemental}

The prior version of this paper included supplemental experiments (unknown-defense settings, no-feedback settings, cross-defense transfer, threshold sweeps, and held-out defense validation) conducted under the string-matching proxy methodology. Because these proxy results \emph{contradict} our native end-to-end findings (e.g., proxy reported Spotlighting ASR=1.00; native shows ASR=0.21), we have removed them from this revision rather than present unsupported numbers. We summarize what was removed and what remains:

\paragraph{Removed (proxy-based, not re-run natively).} (1) Settings A/B/C (unknown-defense, no-feedback, cross-defense transfer)---all used proxy ASR. (2) Threshold sweep for Task Shield and Attention Tracker---used proxy FPR/USR metrics; the native pipeline does not use the same threshold mechanism. (3) Held-out defense validation via SecAlign---used proxy ASR on Mistral-7B. (4) Query/cost analysis referencing the 3{,}900-trial proxy evaluation.

\paragraph{Retained (native end-to-end).} (1) Main results table (\Cref{tab:main}): 10 defenses plus no-defense across L0/L1/L2 (330 benign + 990 static + 990 matched-suffix = 2{,}310 main trials), native \texttt{security\_from\_traces} + \texttt{user\_task.utility}. (2) P2 ablation (\Cref{sec:p2_ablation}, \Cref{app:p2_ablation}): 270 trials, three configurations, giving 2{,}580 total. (3) Over-blocking diagnosis (\Cref{app:overblock}): benign L0 baseline comparison. (4) Statistical analysis (\Cref{app:stats}): marginal Wilson CIs plus paired McNemar and cluster bootstrap for L1-vs-L2 differences.

\paragraph{Capability-monotonic attack set.} Our L2 condition is a candidate set that always contains the exact static (L1) payload plus a defense-matched suffix, a generic authority suffix, a length-matched control, and $k$ paraphrases; we report the worst-case (best-of-$k$) Robust ASR $=\max(\mathrm{ASR}_{L1},\mathrm{ASR}_{L2\text{-set}})$, which is $\geq \mathrm{ASR}_{L1}$ by construction (\Cref{app:native}). The length-matched control isolates whether a matched suffix's effect is due to content or merely to added length.

\paragraph{Planned.} A matched-vs-mismatched transfer matrix and search-/optimization-based adaptive attackers (beyond our fixed candidate set) are planned future work; the native pipeline infrastructure (\Cref{app:native}) supports them, and the current constraint is API cost and time. Broader coverage (additional backbones and AgentDojo suites) is likewise future work.
